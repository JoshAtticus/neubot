Rate Limits API

Endpoint

GET /api/limits

Headers

  • Cookie: Required. Must include session=... (captured from browser login).
  • Accept: application/json

Description

Retrieves the current usage and limits for the user (or IP address if guest).

Note: Total query limits have been removed. Limits only apply to specific tools.

Response Schema

{
  "search": {                    // Limits for 'search' actions (Brave API)
    "used": int,                 // Requests made this month
    "limit": int,                // Max requests allowed
    "remaining": int             // limit - used
  },
  "weather": {                   // Limits for 'weather' actions (OpenWeatherMap)
    "used": int,
    "limit": int,
    "remaining": int
  },
  "total": {                     // DEPRECATED. Legacy field.
    "used": int,
    "limit": int,
    "remaining": int
  },
  "reset": {                     // Reset schedule information
      "started": boolean,        // Whether usage tracking has started
      "timestamp": int|null,     // Unix timestamp of next reset
      "days_remaining": int|null,// Days until next reset
      "date": string|null        // YYYY-MM-DD string of next reset
  }
}
