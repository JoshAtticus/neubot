Authentication API

Neubot uses OAuth2 for authentication with Google and GitHub.

Authentication Types

Neubot currently primarily supports Session-based Authentication (Cookies). There is no persistent API Key mechanism for third-party applications at this time.

Integration for Third-Party Apps

To authenticate a custom application (e.g., mobile app, CLI) against the Neubot API:

  1. Open a webview or browser window pointing to /api/auth/login/google (or github).
  2. Allow the user to complete the OAuth flow in the browser.
  3. Upon successful login, the server will set a session cookie on the response.
  4. Capture and store this cookie.
  5. Include this session cookie in the Cookie header of all subsequent API requests.

Login Endpoints

GET /auth/login/google

Redirects the user to Google's OAuth consent screen.

GET /auth/login/github

Redirects the user to GitHub's OAuth consent screen.

Logout

GET /auth/logout

Clears the user session and redirects to the home page.

Flow

  1. User clicks login button in UI (hitting /login -> views.login_page).
  2. Frontend redirects to one of the endpoints above.
  3. Server redirects to Provider.
  4. Provider redirects back to /auth/google or /auth/github with a code.
  5. Server exchanges code for token, creates/updates user in DB, and sets session cookie.
  6. User is redirected to /.
